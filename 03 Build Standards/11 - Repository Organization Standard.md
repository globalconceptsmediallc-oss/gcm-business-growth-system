# Repository Organization Standard

Version: 1.0

Status: Foundation

Purpose:

Establish a consistent organizational structure for every Global Concepts Media repository. This standard ensures repositories remain scalable, understandable, maintainable, and predictable regardless of size or subject matter.

---

## Principle

A repository is an operating system.

Its structure should communicate purpose before a single document is opened.

Consistent organization reduces confusion, accelerates onboarding, simplifies maintenance, and creates permanent organizational memory.

Repository organization should never depend on personal preference.

It should follow documented standards.

---

## Why It Matters

As repositories grow, inconsistent naming and organization increase complexity.

Poor organization creates:

- Duplicate work.
- Lost documentation.
- Broken internal links.
- Confusing navigation.
- Inconsistent standards.
- Higher maintenance costs.

A standardized repository structure ensures every document has a permanent location and every contributor understands where new knowledge belongs.

---

## Standard

Every Global Concepts Media repository will follow a consistent organizational structure.

### Repository Root

The repository root contains only major operating system components.

Example:

README.md

00 First Principles/

01 Beliefs/

02 Growth Framework/

03 Standards/

04 Systems/

05 Metrics/

---

### Folder Naming

Folders:

- Never use file extensions.
- Use Title Case.
- Begin with a numerical prefix.
- Represent major sections of the operating system.

Examples:

00 First Principles

01 Beliefs

02 Growth Framework

03 Standards

---

### File Naming

Every document shall:

- Use Markdown (.md)
- Begin with a numerical prefix.
- Use descriptive names.
- Use Title Case.
- Avoid abbreviations whenever practical.

Examples:

01 - The First Principles of Client Acquisition.md

02 - Opportunities Create Conversations.md

03 - Trust Is Earned Before It Is Requested.md

---

### Numbering

Documents should always be numbered sequentially.

Numbers represent organizational order—not importance.

Additional documents should extend existing numbering rather than disrupt established organization.

---

### Folder Creation

Folders should never exist without documentation.

Creating a folder requires creating its first document simultaneously.

Empty folders are not permitted.

---

### Repository Growth

New documents must answer:

What permanent purpose does this serve?

If no permanent purpose exists, the document should not be created.

Every document must earn its place.

---

## Decision Framework

Before creating a document, ask:

Does this belong in an existing folder?

Is this a Principle?

Is this a Belief?

Is this a Framework?

Is this a Standard?

Is this a System?

Is this a Metric?

Does this improve the operating system?

If the answer is unclear, the repository architecture should be reconsidered before creating the document.

---

## Measurement

This standard is successful when:

- Every document has a permanent location.
- Repository navigation remains intuitive.
- Duplicate documentation decreases.
- Contributors can locate information quickly.
- Repository growth remains organized.
- Standards remain easy to maintain.

---

## Continuous Improvement

Repository organization evolves through experience.

As new operating systems are created, improvements to structure, naming, and documentation should be incorporated into future repositories while preserving consistency across the Global Concepts Media ecosystem.
